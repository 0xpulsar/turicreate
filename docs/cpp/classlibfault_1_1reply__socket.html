Detailed Description

Constructs an (optionally) zookeeper backed reply socket. Will automatically retry sockets. This object is very much single threaded, with the exception of the key management routines. i.e. keyval_change() could be called from a different thread.

This object works together with the socket_receive_pollset(). The general construction is to

  • Create a reply_socket
  • Create a socket_receive_pollset
  • Register the keys this socket should listen on (register_key() )
  • register this reply_socket with the pollset (add_to_pollset())
  • start the pollset ( socket_receive_pollset::start_poll_thread() )
Note
if zookeeper is enabled, The first part of the message must be a key. This must be a key the current client is registered for.

typedef boost::function<bool (zmq_msg_vector& recv, zmq_msg_vector& reply)> libfault::reply_socket::callback_type

Returns true if there are contents to reply. Returns false otherwise. If the reply socket is connected to a request socket, this must always return true.

Note
There is no provided way to figure out if a reply is necessary. This must be managed on a higher protocol layer.

Constructs a reply socket.

If "keyval" is NULL, the alternate_bind_address MUST be a valid zeromq endpoint address. In which case, the reply_socket will bind to that address.

Parameters
zmq_ctxA zeroMQ Context
keyvalA zookeeper key_value object to bind to (may be NULL)
callbackThe function used to process replies.
alternate_bind_addressIf set, this will be address to bind to. Otherwise, binds to a free tcp address.
